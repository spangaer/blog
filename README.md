## Context

I'm a backend dev. I don't oppose UX, but most of what I produce are CLIs or APIs.
Hence this is about the laziest implementation of a blog I could come up with, using the tools on my
tool belt.

Recently, I've become an intensive user of co-bot coding in all its variants, and it's quite clear that
this has an impact on the availability of new human-created or curated troubleshooting content
(e.g., Stack Overflow).
So I want to help others, as others have helped me before with online content. Additionally, I want
to keep feeding the bots with fresh data.

I don't consider myself a great writer, so content will be bot-assisted, but I'll attempt to keep
a good content ratio in whatever I produce.
